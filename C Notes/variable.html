Variable in C Language

In the C programming language, a variable is a named storage location in memory that can hold data. Variables are used to store and manipulate different types of data such as integers, floating-point numbers, characters, and more. Each variable has a data type that specifies what kind of data it can hold and a unique name that you use to reference it in your code.
Here's the basic syntax for declaring a variable in C:
data_type variable_name;

data_type: This specifies the type of data the variable can hold, such as int for integers, float for floating-point numbers, char for characters, etc.

variable_name: This is the name you give to the variable, which must follow C's naming rules and conventions.

For example, you can declare an integer variable named age like this:

int age;

You can also initialize the variable when you declare it. For example:

int age = 25;

After declaring and initializing a variable, you can assign values to it, read its value, and perform various operations using that variable in your C program. Variables are an essential part of C programming as they allow you to work with and manipulate data within your code.

In the C programming language, values of variables can be changed. But sometimes, we want a value to be fixed. Here comes the CONSTANTS. In C programming language, constants are values that do not change during the execution of a program. Constants are used to represent fixed values, such as numbers, characters, or strings, that remain constant and do not vary.

To make a value constant, we need to add 'const' keyword before its declaration. For example:

const int pi = 3.14;

Enumeration Constants: These are user-defined constants created using the enum keyword. For example:

enum pi = { RED, GREEN, BLUE };

Symbolic Constants: These are created using the #define preprocessor directive and are used to give a name to a constant value. For example:

#define PI 3.14159265359;

Constants are used in C to make programs more readable, maintainable, and flexible. By using symbolic constants and enumerations, you can give meaningful names to values, making the code more self-explanatory and easier to change in the future if needed.

There are some rules defined for naming variables. We can use anything or any name we want as variable name. The general rules for naming variables are:

  • Names can contain letters, digits and underscores
  • Names must begin with a letter or an underscore (_)
  • Names are case sensitive (myVar and myvar are different variables)
  • Names cannot contain whitespaces or special characters like !, #, %, etc.
  • Reserved words (such as int) cannot be used as names
